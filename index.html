Gestion des Certificats

Contexte

Ecocert a déjà un site web réalisé avec le Framework CodeIgniter en langage PHP 7, avec une base de données MariaDB.

Ecocert souhaite enrichir ses services en offrant au grand public, la possibilité de visualiser en ligne les certificats délivrés à ses clients.

Un salarié d’Ecocert doit pouvoir enregistrer les informations concernant les clients et leurs activités ainsi que les certifications dans la base de données « gestion_certificat ».
Un internaute doit pouvoir visualiser les certificats souhaités selon différents critères de recherche. Certains certificats sont directement consultables en ligne, d’autres nécessitent une demande au préalable auprès d’Ecocert. Cette demande doit pouvoir être formulée depuis le site web.

Veille Technologique

Définition

La veille technologique, c’est la collecte, le partage et la diffusion d’informations, permettant d’anticiper ou de s’informer sur des changements en matière de Recherche, Développement, Brevet, Lancement de nouveaux produits, matériaux, processus, concepts, innovation de fabrication, etc…, afin d’évaluer l’impact sur l’environnement et l’organisation.

API Discord

Discord est un réseau social initialement conçu pour les "gamers". Discord a mit en place une API pour permettre à ses utilisateurs de créer des bots pour les aider à gérer leurs groupes de jeux (appelés "serveurs"). L'API permet aussi de faire un système de connexion sur notre site web personnel avec leur "Log In With Discord". Cela fonctionne sur le même principe que les connexions via un compte Google, sauf qu'on se connect via un compte Discord.

Evolution de l'API

Il y a plusieurs versions de l'API utilisables. La version 6 est celle par défaut, la version 8 et la version 9 sont les versions les plus récentes. Quand j'ai commencé ma veille, la version 8 venait de sortir.

24/09/2020 API V8
15/12/2020 SLASH COMMANDS
22/01/2021 PERMISSION CHECKING
09/02/2021 SLASH COMMANDS DMs
05/04/2021 SLASH COMMAND PERMISSION
28/04/2021 API V9
26/05/2021 MESSAGE COMPONENTS

La plus part des chemins ont été remplacé (les traits d'union sont remplacés par des underscores)

Une nouvelle fonctionnalité implémenté : les slash commands. Avant pour faire une commande pour un bot il fallait tout simplement écrire la commande dans un salon textuel et l'envoyer comme un message normal. Maintenant on préfix la commande avec un slash (/) et quand on l'envoit, la commande n'apparaît pas dans le salon mais le bot la reçoit quand même. C'est plus joli et surtout ça fait moins de notifications pour les autres membres du serveur (moins de spam).

Les bots peuvent créer des salons sur les serveurs. Le problème, c'est qu'ils pouvaient créer des salons et leur donner des permissions qu'eux même n'avaient pas. Cela pouvait être utilisé pour de l'escalade de privilèges ainsi le bot pouvait s'octroyer plus de droit qu'il n'en avait à la base.

Les slash commands sont maintenant disponible en messages privés. Avant on ne pouvait les utiliser que sur des serveurs. Maintenant on peut les utiliser dans les conversations privés avec un bot.

La permission "slash command" a été ajouté dans les serveurs. Maintenant on peut autoriser les membres d'un serveur à utiliser les slash commands ou non.

La nouvelle version de l'API supporte maintenant les "threads". Cette fonctionnalité permet de répondre à un message un peut comme sur messenger. Les bots peuvent maintenant le faire aussi mais uniquement sur la version 9 de l'API.

Les bots peuvent maintenant ajouter des "composents" à leurs messages. Pour l'instant il n'existe que deux types de composents : les conteneurs qui peuvent contenir d'autres composents (un conteneur ne peut pas contenir d'autres conteneurs) et les boutons. Ces composents rendent les messages des bots intéractifs.
